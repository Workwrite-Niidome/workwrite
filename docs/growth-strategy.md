# Workwrite 成長戦略書

## 1. 市場概況

### 日本電子書籍・Web小説市場

| 指標 | 数値 |
|------|------|
| 電子書籍市場規模（2024年） | 6,703億円 |
| 予測市場規模（2028年） | 約8,000億円 |
| Web小説プラットフォーム月間利用者 | 推定3,000万人超（小説家になろう単体で700万人/月） |
| 年間成長率 | 約5-7% |

### 主要競合プラットフォーム

| プラットフォーム | 月間ユーザー | 強み | 弱み |
|----------------|------------|------|------|
| 小説家になろう | 700万+ | 圧倒的ユーザー数、書籍化パイプライン | UI老朽化、ランキング偏重、執筆支援なし |
| カクヨム（KADOKAWA） | 200万+ | 出版社直結、レビュー文化 | KADOKAWA色が強い、ジャンル偏り |
| アルファポリス | 150万+ | 収益化しやすい、書籍化実績 | 独占契約要求、UI古い |
| エブリスタ | 100万+ | スマホ特化、女性ユーザー多い | 短編偏重、収益化が弱い |
| novelpia/Novelup | 新興 | 韓国系の新しいUI | 日本市場でのブランド力不足 |

### 既存プラットフォームに対する作者の不満（市場機会）

1. **ランキング至上主義** — 毎日更新しないとランキングから消える「更新圧」
2. **フィードバック不足** — PVとブックマーク数しか見えない。何が良くて何が悪いか分からない
3. **執筆支援の欠如** — プロット管理、キャラ管理、世界観管理は全て外部ツール頼り
4. **発見されにくい** — 新規作者は既存ランキングの壁を超えられない
5. **収益化の壁** — 書籍化されない限り収入ゼロ（なろう）、または微々たる広告収入
6. **読者との断絶** — コメント欄以外の読者接点がない

## 3. ターゲットセグメント戦略

### Phase 1 ターゲット: 「品質志向の中堅作者」（最初の6ヶ月）

**ペルソナ:**
- 既存プラットフォームで10万字以上書いた経験あり
- ランキング競争に疲弊、または伸び悩み
- 執筆の質を上げたいがフィードバックがない
- 年齢25-40歳、社会人の趣味作者が中心

**なぜこの層から始めるか:**
- 既に執筆習慣がある → 定着率が高い
- 品質への意識が高い → Workwriteの価値を最も感じる層
- 発信力がある → 口コミの起点になる
- なろう/カクヨムに不満を持っている → スイッチコスト低

**獲得目標:** 500人のアクティブ作者

### Phase 2 ターゲット: 「感想を深く伝えたい読者」（6-12ヶ月）

**ペルソナ:**
- 作品の感想をSNSに長文で書くタイプ
- 「ここが好き」を具体的に伝えたい
- AIコンパニオンでキャラと対話したい

**獲得目標:** 5,000人のアクティブ読者

### Phase 3 ターゲット: 「初めて書いてみたい層」（12-18ヶ月）

**ペルソナ:**
- 小説を書いてみたいが何から始めればいいか分からない
- Creation Wizardのガイドがあれば書ける
- AI支援で最初の壁を越えられる

**獲得目標:** 2,000人の新規作者

## 4. 成長戦略（具体的施策）

### 戦略1: 作者コミュニティ浸透（最重要・最速）

#### 1-1. X（Twitter）作者クラスタ攻略

日本のWeb小説作者はX上で強いコミュニティを形成している。ハッシュタグ `#小説書きさんと繋がりたい`、`#物書きさんと繋がりたい` は常時活発。

**施策:**
- **週次「AI品質分析レポート」投稿**: 匿名化した実際のスコアリング結果を見せる。「あなたの作品の『没入感』を数値化します」
- **「Before/After」事例**: AI執筆アシストを使った改稿の具体例（作者許諾済み）
- **作者インタビュー**: 早期ユーザーの体験記を定期投稿
- **ハッシュタグキャンペーン**: `#Workwriteで書いてみた` — 初回無料クレジット20crで体験

**KPI:** 月間インプレッション10万、フォロワー1,000人（3ヶ月目標）

#### 1-2. 創作系Discord/Slackコミュニティへの参入

- 大規模創作Discordサーバー（創作クラスタ、物書きの集い等）でのプレゼンス
- Workwrite公式Discordサーバー開設 → 作者同士のフィードバック交換の場
- 「執筆もくもく会」（オンライン作業会）の定期開催

#### 1-3. note/はてなブログでの発信

- 「Web小説のプロット設計術」「キャラクターの口調を統一する方法」等、創作ノウハウ記事
- 記事末尾でWorkwriteのCreation Wizardを紹介（自然な導線）
- SEOで「小説 書き方」「プロット 作り方」等の検索流入

### 戦略2: 既存プラットフォームからの移行支援

#### 2-1. インポート機能

- **なろう/カクヨムからのワンクリックインポート**: APIまたはスクレイピングで既存作品を移行
- インポート後に自動でAI品質分析を実行 → 「あなたの作品の強みと改善点」レポート提供
- これ自体が強力なフック: 「無料であなたの作品を分析します」

#### 2-2. 並行掲載サポート

- 「Workwriteで書いて、なろうにも同時投稿」ワークフロー支援
- 作者のリスクを最小化（既存読者を失わずにWorkwriteも試せる）
- Workwriteで得たAIフィードバックで改稿 → なろう版にも反映

### 戦略3: 口コミドリブン成長

#### 3-1. 作品共有の仕組み

- 作品ページのOGP最適化（X/LINEでシェアした時に美しいカード表示）
- 「この作品の感情体験」共有カード — 感情タグを視覚化したシェア画像自動生成
- 読了後の「この作品を広めませんか？」CTA

#### 3-2. リファラルプログラム

- 招待した作者が初作品を公開 → 招待者に50cr付与
- 招待した読者が初レビュー → 招待者に10cr付与
- 双方にメリットがある設計

#### 3-3. 品質バッジ制度

- AIスコアリングで高評価の作品に公式バッジ付与
- 「Workwrite Quality Certified」的なブランディング
- バッジ付き作品はDiscoverで優先表示

### 戦略4: コンテスト・イベント戦略

#### 4-1. 月次テーマコンテスト

- 毎月テーマを設定（例：「再会」「秘密」「最後の手紙」）
- AI品質スコアリングを審査の一要素に（透明性）
- 入賞作品には大量クレジット付与 + Discover特集掲載
- 短編（5,000字以下）で参加ハードルを下げる

#### 4-2. 「AIと共作」チャレンジ

- AIアシストを積極活用した作品のコンテスト
- 「AIをどう使ったか」のプロセス共有を義務化 → 教育コンテンツ化
- Workwriteならではのユニークなイベント

#### 4-3. 季節イベント

- 年末年始「書き初めチャレンジ」（1月中に1作品完結）
- GW「短編マラソン」
- 夏の「ホラー特集」、秋の「読書の秋コンテスト」

### 戦略5: SEO・コンテンツマーケティング

#### 5-1. 検索流入の獲得

**ターゲットキーワード:**
- 「小説 書き方 AI」（月間検索 推定500-1000）
- 「プロット 作り方 ツール」（月間検索 推定300-500）
- 「Web小説 プラットフォーム 比較」（月間検索 推定1000+）
- 「キャラクター設定 テンプレート」（月間検索 推定500+）
- 「小説 品質 チェック」（月間検索 推定200-300）

**施策:**
- ブログセクション追加（`/blog`）
- Creation Wizardの一部機能を無料公開ツール化（リード獲得）
- 「Web小説プラットフォーム比較2026」等の記事

#### 5-2. YouTube/動画コンテンツ

- 「AIを使った小説執筆の新しい形」デモ動画
- Creation Wizardの使い方チュートリアル
- 作者インタビュー動画（ストーリーテリング）

### 戦略6: パートナーシップ

#### 6-1. 出版社・編集者連携

- Workwriteの品質スコアを出版社のスカウト基準に

#### 6-2. 創作系教育機関

## 8. リスクと対策

| リスク | 影響度 | 対策 |
|--------|--------|------|
| AIコスト増大 | 高 | クレジット制で制御済。Haiku活用でコスト最適化 |
| 既存プラットフォームの模倣 | 中 | 先行者優位を確立。構造化データ統合は模倣困難 |
| 作品数不足で読者が来ない | 高 | インポート機能で初期コンテンツ確保。作者を先に集める |
| サーバー負荷 | 中 | Railway/Vercelのスケーラビリティ。段階的拡張 |
| 品質スコアへの批判 | 中 | スコアは「改善のヒント」として訴求。ランキング排他には使わない |
| 法的リスク（AI著作権） | 低 | AIはあくまでアシスト。著作権は作者に帰属を明記 |

## 9. 競合が追いつけないWorkwriteの堀（Moat）

1. **構造化データの蓄積**: Creation Wizardで蓄積される世界観・キャラ・プロットデータは、他プラットフォームには存在しない。これがAI品質の源泉
2. **品質スコアリングの精度**: 作品数が増えるほどスコアリングの基準が洗練される（ネットワーク効果）
3. **感情タグエコシステム**: 読者の感情フィードバックが蓄積されるほど、推薦精度が向上
4. **AIコンパニオンの体験**: 構造化データがあるからこそ可能な、キャラクターとの対話体験
5. **作者の移行コスト**: Creation Wizardに蓄積されたデータは他プラットフォームに持ち出せない（ロックイン）

## まとめ

Workwriteの成長の鍵は「**品質志向の作者コミュニティを最初に形成する**」こと。なろう/カクヨムの「量と更新頻度」の競争に疲弊した作者層に、「AIが執筆パートナーになる品質重視の場」として訴求する。

作者が集まれば作品が生まれ、作品が生まれれば読者が来る。読者の感情フィードバックが作者のモチベーションになり、さらに良い作品が生まれる。このフライホイールを回すことがWorkwriteの成長エンジンとなる。

```
作者が来る → 良い作品 → 読者が来る → 感情フィードバック → 作者のモチベーション → さらに良い作品 → ...
     ↑                                                                                    │
     └──────────────────── AI品質分析・執筆支援 ──────────────────────────────────────────────┘
```
