# Web小説、縦書きで読みたくないですか？ — Workwriteが縦書きリーダーを作った理由

## 横書きが「当たり前」になった世界で

Web小説を読むとき、文字が左から右に流れていくことに違和感を覚えたことはないでしょうか。

ブラウザで読む以上、横書きは仕方がない。そう思って、私たちもずっと横書き前提でリーダーを設計してきました。でも、日本語の小説は本来、縦書きで読むものです。文庫本を開けば文字は上から下へ流れ、ページは右から左へめくる。その体験に慣れた身体が、横書きのWeb小説にどこか馴染みきれないのは、自然なことかもしれません。

「縦書きで読めたらいいのに」

そんな声をいただいたことをきっかけに、私たちは本気で縦書きリーダーの実装に取り組みました。

## なぜ縦書きにこだわるのか

縦書きと横書きでは、読書体験そのものが変わります。

まず、**視線の動き**が違います。横書きでは視線が左右に往復しますが、縦書きでは上から下への自然な流れになる。日本語話者にとっては、縦の流れのほうが文章のリズムを掴みやすいという研究もあります。

次に、**没入感**。紙の本に近い見た目になることで、「画面で情報を読んでいる」感覚から「本を読んでいる」感覚に近づきます。特に文芸作品や純文学寄りの作品では、この差が大きい。

そして、**ページめくりの感覚**。縦書きでは左方向にページが進みます。スマホやタブレットで左にスワイプしてページをめくる動作は、紙の本をめくる動作と同じ方向。身体に染みついた読書のリズムが、そのまま使えるのです。

## CSSの一行で始まる、終わらない戦い

Webで縦書きを実現するCSS自体は、実はとてもシンプルです。

```css
writing-mode: vertical-rl;
```

たった一行。これだけで文字は縦に並び、右から左へ流れます。

「なんだ、簡単じゃないか」と思われるかもしれません。私たちも最初はそう思いました。しかし、**実用レベルの縦書きリーダー**に仕上げるまでには、想像以上の壁がありました。

### 壁1: ページ境界で文字が切れる

縦書きのテキストは横方向にあふれます。画面幅ごとに「ページ」として区切りたいのですが、単純に画面幅で分割すると、ページの端で文字が途中で切れてしまいます。

画面幅が600pxで、テキストの行が602px目から始まっていたら？ その行は次のページにあるべきなのに、2pxだけ前のページに見切れてしまう。

この問題は、CSSの**マルチカラムレイアウト**を活用することで解決しました。ブラウザに「この幅でカラムを分割してくれ」と伝えると、ブラウザが文字の切れ目を自動で調整してくれます。ページ境界で文字が途切れることはありません。

### 壁2: ブラウザごとに違うスクロール方向

`writing-mode: vertical-rl` を設定すると、コンテンツは右から左に流れます。JavaScriptでスクロール位置を制御するとき、`scrollLeft` という値を使うのですが、この値の**符号がブラウザによって違う**のです。

ChromeやSafariでは正の値、Firefoxでは負の値。同じ「左に100pxスクロール」でも、あるブラウザでは `+100`、別のブラウザでは `-100` になる。

私たちは初期化時にブラウザの挙動を一度だけ検出し、その結果をキャッシュすることで、すべてのブラウザで統一された操作感を実現しています。

### 壁3: スマホでの操作設計

PCではマウスホイールやクリックでページを送れますが、スマホではタッチ操作が主です。ここで難しいのが、**「スワイプ」と「タップ」の区別**。

画面を触って指を動かしたとき、それは「次のページに行きたい」スワイプなのか、それとも「画面の左側をタップした」ページ送りなのか。指の移動距離が小さく、時間が短ければタップ。横方向に大きく動いていればスワイプ。この判定を適切に行わないと、「タップしたのにスワイプ扱いになる」「スクロールしたいだけなのにページが進む」といった不快な体験になります。

## 自分のサイトで縦書きを試してみる

もし自分のWebサイトやブログで縦書きを試したいなら、最低限のCSSはこれだけです。

```css
.vertical-text {
  writing-mode: vertical-rl;
  -webkit-writing-mode: vertical-rl;
  overflow-x: auto;
  height: 80vh;
  padding: 2rem;
}
```

これで、`.vertical-text` クラスを付けた要素の中身が縦書きになります。`height` でコンテナの高さを固定し、`overflow-x: auto` で横スクロールを有効にすれば、長い文章も読めます。

ただし、これだけでは「読書体験」としては不十分です。ページ送り、進捗表示、文字サイズ調整、モバイル対応……。快適な読書のために必要な要素は、CSSの`writing-mode`だけでは実現できません。

## Workwriteの縦書きリーダー

Workwriteでは、リーダーの設定画面から「組方向」を切り替えるだけで、すべての作品を縦書きで読むことができます。

**主な特徴：**

- **ページ送り方式**: 自由スクロールではなく、ページ単位で確実に切り替わる設計。半端な位置で止まることがありません
- **複数の操作に対応**: タップ（画面の左右）、スワイプ、マウスホイール、キーボード矢印キー。どの操作でもページを送れます
- **進捗表示**: フッターにページ番号とプログレスバーを常時表示。全体のどこを読んでいるか一目でわかります
- **横書きにいつでも戻せる**: フッターのボタンからワンタップで横書きモードに戻れます
- **読書機能はそのまま**: ギフトレターやキャラクタートークなど、Workwriteの他の機能も縦書きモードのまま使えます
