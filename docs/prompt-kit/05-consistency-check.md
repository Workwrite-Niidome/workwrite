# プロンプト5: 一貫性チェック

全話を書き終えたら、最後にこのプロンプトで品質チェックを行います。

---

```
あなたは小説の校閲の専門家です。
以下の小説の全話を読み、一貫性の問題を指摘してください。

【キャラクター設定（照合用）】
（プロンプト1の出力を貼り付け）

【世界設定（照合用）】
（世界のルール・制約を貼り付け）

【全話のテキスト】
（全話の本文を貼り付け。長い場合は各話の冒頭・中盤・末尾を抜粋）

以下の観点でチェックし、問題があれば具体的に指摘してください:

1. 【キャラクターの一人称】
   各キャラクターの一人称が全話を通じて一貫しているか。
   ブレている箇所があれば「第○話で○○が『△△』と言っているが、設定は『□□』」と指摘。

2. 【キャラクターの口調】
   口調設定と実際のセリフが一致しているか。
   逸脱があれば具体的に引用。

3. 【前話との矛盾】
   各話の冒頭が前話の末尾から自然に繋がっているか。
   矛盾があれば「第○話末尾で○○だが、第△話冒頭では□□になっている」と指摘。

4. 【存在しない事実の参照】
   それ以前の話に書かれていない出来事を「あった」として参照していないか。
   ハルシネーションがあれば「第○話で『○○と約束した』とあるが、そのような場面は存在しない」と指摘。

5. 【世界設定違反】
   世界設定に存在しない技術・概念が登場していないか。

6. 【伏線の回収】
   設置された伏線が回収されているか。未回収の伏線をリストアップ。

問題がなければ「✅ 問題なし」と明記してください。
```

## このキットのまとめ

| プロンプト | やること | 頻度 |
|-----------|---------|------|
| 01 キャラクター設計 | 登場人物を詳細に設計 | 最初に1回 |
| 02 プロット構築 | 三幕構成 + 伏線計画 | 最初に1回 |
| 03 章立て設計 | 各話の詳細設計 | 最初に1回 |
| **04 各話の執筆** | **毎話の本文生成** | **毎話** |
| 05 一貫性チェック | 全体の品質検証 | 最後に1回 |

**04の「各話の執筆」が一番重要で、一番面倒です。**
毎話ごとに前話の末尾を更新し、キャラ設定を含め、伏線を管理する必要があります。
