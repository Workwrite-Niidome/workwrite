# Workwrite 小説構造化プロンプトキット

**ChatGPT / Claude / Gemini など、どの生成AIでも使えるプロンプト集です。**

Workwriteが内部で行っている「小説の構造化」を、手動で体験できます。
5つのプロンプトを順番に使うことで、キャラクター設計からエピソード執筆まで一貫した物語を作れます。

## 使い方

1. `01-character-design.md` → キャラクター設計
2. `02-plot-structure.md` → プロット構築
3. `03-chapter-outline.md` → 章立て設計
4. `04-episode-writing.md` → 各話の執筆（メイン）
5. `05-consistency-check.md` → 一貫性チェック

各ファイルの中のプロンプトをコピーして、ChatGPTやClaudeに貼り付けてください。
前のステップの出力結果を、次のステップの入力に含めることで、物語の一貫性が保たれます。

## なぜ構造化が必要なのか

AIに「小説の続きを書いて」と頼むと、こんな問題が起きます:

- キャラクターの口調が突然変わる
- 前の話で起きていないことを「あった」として書く
- 世界設定にない技術が突然登場する
- 伏線を張ったのに回収されない

これらは**AIに物語の構造が伝わっていない**ことが原因です。
このプロンプトキットは、AIに構造を伝えるための「型」を提供します。

## Workwriteならこれが全自動

手動でプロンプトキットを使ってみると、「毎話これをやるのか...」と感じるはずです。

**[Workwrite](https://workwrite.jp)** では、この構造化プロセスを自動化しています:
- キャラクター設定を保存すると、以降の執筆でAIが自動参照
- 各話の保存時に構造分析が自動実行
- 前話の情報が自動でコンテキストに注入
- 伏線の設置・回収を自動追跡
- 6軸のAI品質スコアリングで作品の強みを可視化
