# Workwrite スコアリングシステム 詳細解説

## 概要

Workwriteのスコアリングシステムは、小説作品の品質を**6つの評価軸**で多角的に分析し、0〜100点のスコアを算出するシステムです。プログラム的なテキスト解析（無料）とLLM（Claude Haiku）による文学的評価を組み合わせた**2フェーズパイプライン**で動作します。

---

## 評価軸（6軸・各0〜100点）

### 1. 没入力（Immersion）
読者を物語に引き込む力。

| 評価項目 | 内容 |
|----------|------|
| 冒頭の引き | 最初の数段落で注意を捕捉できるか |
| 五感描写 | 視覚・聴覚・触覚などの描写の豊かさ |
| 文体のリズム | 短文と長文の緩急 |
| 情報バランス | 説明過多/不足でないか |
| シーン転換 | 適切なペーシング |
| 会話と地の文のバランス | 偏りがないか |

### 2. 変容力（Transformation）
読後に読者の感情・価値観を変える力。

| 評価項目 | 内容 |
|----------|------|
| テーマの一貫性 | 一つの明確なテーマが貫かれているか |
| 感情の軌跡 | エピソードごとの起伏 |
| クライマックスのインパクト | 感情のピークが「earned」か |
| 余韻 | 読後に残る問い・感情 |
| 普遍性 | 時代・文化を超える共感性 |

### 3. 拡散力（Virality）
人に薦めたくなる・語りたくなる力。

| 評価項目 | 内容 |
|----------|------|
| 独自性 | 「この作品にしかない」要素 |
| バイラルフック | 人に話したくなるシーン |
| 記憶に残る瞬間 | 読後も残り続ける場面 |
| 感情の動かし方 | ジャンルに応じた感動（興奮/静かな衝撃/恐怖） |
| 導入の魅力 | 冒頭が続きを読ませるか |

### 4. 世界構築力（World Building）
作品世界のデザイン品質。

| 評価項目 | 内容 |
|----------|------|
| 設定のレイヤー | 地理・魔法・社会・技術・文化の5層 |
| 内部整合性 | 矛盾がないか |
| 独自ルール | 魔法体系・社会制度の論理性 |
| 自然な統合 | 設定が地の文に溶け込んでいるか（info dumpしていないか） |
| 語彙の豊かさ | 多様でオリジナルな言語表現 |
| ジャンル適合 | ライトな恋愛物とエピックファンタジーで基準が異なる |

### 5. キャラクター深度（Character Depth）
登場人物の造形と成長。

| 評価項目 | 内容 |
|----------|------|
| 声の一貫性 | 台詞でキャラが区別できるか |
| 動機の明確さ | 行動の理由が読者に伝わるか |
| 内面の深さ | 感情・思考・葛藤の描写 |
| 成長弧 | 物語を通じたキャラクターの変化 |
| 差別化 | 多人数でも全員が区別可能か |
| 関係性の複雑さ | 善悪二元論を超えた人間関係 |

### 6. 構造スコア（Structural Score）
プロットの設計品質。

| 評価項目 | 内容 |
|----------|------|
| 完全な弧 | 起承転結が見えるか |
| 伏線処理 | 張った伏線が回収されているか（意図的な未回収含む） |
| ペーシング | 緊張と緩和の緩急 |
| クライマックス品質 | 真の山場が存在するか |
| 結末 | 適切なクロージング |
| 設計意図の遂行 | 設計データがある場合、それが実行されたか |

## 総合スコアとティア

**総合スコア = 6軸の等加重平均（0〜100）**

| スコア範囲 | ティア | 意味 |
|-----------|--------|------|
| 90〜100 | 傑作 | 極めて稀。出版レベルを大きく超える |
| 80〜89 | 秀作 | 出版水準。完成度が高い |
| 65〜79 | 良作 | 明確な強みがある |
| 50〜64 | 佳作 | Web小説標準レベル |
| 30〜49 | — | 改善の余地が大きい |
| 0〜29 | — | 大幅な改善が必要 |

> **設計方針**: 全作品の平均が60点前後になるようキャリブレーション。70点以上は全体の上位層。30点台も品質に応じて正当に出す（スコアインフレの防止）。

## 2フェーズ パイプライン

### Phase 1: プログラム的テキスト解析（LLMコスト0）

LLMに渡す前に、テキストから以下のメトリクスを自動抽出します。

#### スケール指標
- `totalCharCount` — 全エピソード合計文字数
- `episodeCount` — エピソード数
- `avgEpisodeLength` — エピソードあたり平均文字数

#### 会話分析
- `dialogueRatio` — 「」括弧内テキストの割合（0.0〜1.0）
- `dialogueLineCount` — 会話文の総数
- `avgDialogueLength` — 会話文あたり平均文字数

#### 文構造
- `avgSentenceLength` — 文あたり平均文字数
- `sentenceLengthVariance` — 文長の分散
- `shortSentenceRatio` — 20文字未満の短文の割合
- `longSentenceRatio` — 80文字超の長文の割合

#### 段落構造
- `avgParagraphLength` — 段落あたり平均文字数
- `paragraphCount` — 段落総数
- `singleLineParagraphRatio` — 1行段落の割合

#### 語彙
- `uniqueKanjiCount` — ユニーク漢字数
- `vocabularyRichness` — Type-Token Ratio（語彙の豊かさ）

#### シーンとペース
- `sceneBreakCount` — シーン転換数
- `exclamationDensity` — 感嘆符密度（/1000字）
- `questionDensity` — 疑問符密度（/1000字）
- `ellipsisDensity` — 三点リーダー密度（/1000字）

#### エピソード間分散
- `episodeLengthVariance` — エピソード長の統計的分散
- `dialogueRatioByEpisode` — エピソードごとの会話率

---

### Phase 2: LLMスコアリング（Claude Haiku）

Phase 1で得たメトリクスと、戦略的に抽出した**4つのテキストサンプル**をLLMに送信します。

#### テキストサンプル戦略

作品の長さに応じてサンプルサイズが適応的に変化します。

| 作品長 | サンプルサイズ/箇所 | カバー率 |
|--------|-------------------|---------|
| 短編（〜2万字） | 2,000字 | 約10% |
| 中編（2〜5万字） | 3,000字 | 約6% |
| 長編（5〜10万字） | 4,000字 | 約4% |
| 超長編（10万字〜） | 5,000字 | 約2〜5% |

#### サンプル抽出位置（4箇所）
1. **冒頭** — 第1エピソードの開始部分
2. **中盤** — 作品中盤のエピソード
3. **クライマックス付近** — 全体の約75%地点
4. **結末** — 最終エピソードの末尾

#### 長編作品モード（15万字超）
- 全エピソードにまたがる分散サンプリング
- 以下の観点を特別に考慮：
  - 伏線のスパン（序盤→終盤の回収）
  - キャラクターの成長（初回→最終回）
  - テーマの深化
  - 文体の成熟

## 構造データの収集

スコアリングはテキストだけでなく、作品に紐づく設計データも参照します。

### 参照されるデータソース

| ソース | 提供する情報 |
|--------|-------------|
| `EpisodeAnalysis` | エピソード要約、感情弧、POV一貫性 |
| `Foreshadowing` | 伏線の張り/回収カウント、回収率 |
| `WorldSetting` | 世界設定（地理/魔法/社会/技術/文化の5カテゴリ） |
| `CharacterDialogueSample` | キャラクターごとの代表的台詞（最大3件/人） |
| `StoryCharacter` | キャラクター設計（名前/役割/性格/口調/弧） |
| `StoryArc` | 作品設計（前提/葛藤/テーマ/幕構成） |
| `StoryScene` | シーン設計（感情ターゲット/強度）→クライマックス特定 |

### 構造プロファイル出力
- `narrativePOV` — 視点（一人称/三人称限定/神視点）
- `povConsistency` — 視点の一貫性（Boolean）
- `foreshadowingResolutionRate` — 伏線回収率（%）
- `analysisCoverage` — エピソード分析済み割合（0.0〜1.0）
- `hasDesignData` — 構造化された設計データの有無

> **注**: 設計データがない作品（外部インポートなど）でもスコアリングは可能。テキスト解析のみで評価されます。

## エピソード分析（Episode Analysis）

各エピソード公開時に自動トリガーされる構造解析。スコアリングの基礎データになります。

### 抽出される情報

```
summary          — 200〜400字の要約（キーとなる引用含む）
endState         — シーン/感情/状況の終了状態（300〜500字）
narrativePOV     — 視点
emotionalArc     — 感情の流れ（例: 期待→不安→決意）
timelineStart/End — 時系列（翌朝/3日後 等）
locations[]      — 登場場所
characters[]     — 登場人物（名前/役割/行動/状態）
foreshadowings[] — 伏線（plant/develop/resolve）
dialogueSamples[] — 代表的台詞（人物/台詞/文脈/感情）
newWorldRules[]  — 新たに判明した世界設定
characterConsistency[] — 設計との整合性チェック
```

### ルール
- テキストに明示された事実のみ抽出（推測しない）
- `StoryCharacter` の設計データと比較してキャラ一貫性をチェック
- 前エピソードとの連続性を確認
- バージョン管理されており、再分析時にインクリメント

## クレジットコスト

スコアリングはクレジットを消費します。

### コスト計算式
```
コスト = f(入力文字数, システムプロンプト文字数, 最大出力トークン)
  → 文字 → トークン変換 → クレジット変換（AiTierServiceによる）
  → 最低1クレジット
```

### 仕組み
1. スコアリング**実行前**にコスト見積もりを提示
2. ユーザーが確認後に実行
3. 失敗時は自動的にクレジットを**返金**（トランザクション安全性）

---

## LLM出力フォーマット

### 軸ごとの分析
```json
{
  "immersion": {
    "score": 72,
    "analysis": "強み（具体的引用付き1〜2文）→「次のレベル」アドバイス（1〜2文・前向きな表現）"
  },
  ...
}
```

### 改善アドバイス（3件）
1. 最もインパクトが大きいものを最初に
2. 「どこで/どうやって/なぜ」の具体性
3. 作品・作者のスタイルを尊重した提案

### 感情タグ（3〜5個選択）
`勇気` `涙` `世界観` `癒し` `興奮` `思考` `笑い` `共感` `畏怖` `郷愁` `サスペンス` `ミステリー` `希望` `美` `成長`

---

## オリジナリティスコア（別軸）

品質スコアとは別に、AI生成率を追跡するスコアがあります。

```
originality = 1.0 - (AI生成文字数 / 総文字数)
```

| オリジナリティ | 表示色 | 意味 |
|--------------|--------|------|
| 90%以上 | 緑 | ほぼ人間執筆 |
| 70〜89% | 青 | 適度なAI活用 |
| 50〜69% | 橙 | AI使用多め |
| 50%未満 | 灰 | AI主体 |

## API エンドポイント

| メソッド | パス | 説明 |
|---------|------|------|
| `GET` | `/scoring/works/{workId}/estimate` | コスト見積もり（残高・文字数・エピソード数含む） |
| `POST` | `/scoring/works/{workId}` | スコアリング実行（クレジット消費） |
| `GET` | `/scoring/works/{workId}` | スコア取得 |
| `GET` | `/scoring/works/{workId}/analysis` | 分析テキスト付きスコア取得 |
| `POST` | `/scoring/episodes/{id}` | 単一エピソードのスコアリング |

## フロントエンド表示

### ScoreCard（詳細表示）
- 6軸のレーダーチャート
- 総合スコア（0〜100）+ ティアラベル
- 各軸の個別カード（説明付き）
- 感情タグバッジ
- 改善アドバイス表示
- スコア共有機能

### ScoreBadge（コンパクト表示）
- スコア数値 + ティアを1行で表示
- ティアごとに色分け（紫=傑作 / 緑=秀作 / 青=良作 / 橙=佳作 / 灰=ティアなし）

### 共有機能
- X（Twitter）/ LINE / URLコピー
- ティアごとに異なる共有メッセージ

---

## 設計上の重要な判断

1. **公開済み作品のみ**がスコアリング対象（下書きは手動トリガー可）
2. **6軸等加重**で総合スコア算出（伝統的4要素ではない）
3. **Claude Haiku**をLLMに採用（コスト/品質バランス）
4. **システムプロンプトキャッシュ**で再スコアリング高速化
5. **長編は分散サンプリング**（切り捨てではない）
6. **インポート作品の特別処理** — テキストのみで設計データなしでも評価可能
7. **バージョン追跡**による再スコアリング比較
8. **失敗時自動返金**のクレジットモデル
9. **構造データはオプション** — なくてもスコアリング可能
10. **分析カバレッジ指標**で構造分析の進捗を可視化
