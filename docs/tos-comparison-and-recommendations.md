# 利用規約 比較調査 & Workwrite改善提案

## 1. 他サービスとの横断比較

### 重複投稿（マルチポスト）

| サービス | ルール |
|----------|--------|
| 小説家になろう | 事実上許可（明示規定なし） |
| カクヨム | 許可（明示規定なし） |
| アルファポリス | **明示的に許可** |
| エブリスタ | 明確な規定なし |
| ノベルアップ+ | 許可（転載元規約に従う旨を記載） |
| **Workwrite** | **規定なし** → 要追加 |

> **業界標準**: 重複投稿は全サービスで実質許可。禁止するサービスは存在しない。

### 著作権の帰属

| サービス | 帰属先 | サービス側の利用範囲 |
|----------|--------|---------------------|
| なろう | 投稿者 | 運営保守に必要な範囲 |
| カクヨム | 投稿者 | 無償・非独占的ライセンス（掲載・配信・複製・派生物） |
| アルファポリス | 投稿者 | 広告宣伝目的の無償利用 |
| エブリスタ | 投稿者 | **あらゆる方法での利用**（最も広範） |
| ノベルアップ+ | 投稿者 | 運営保守に必要な範囲 |
| **Workwrite** | 投稿者 | プラットフォーム上の表示・配信（非独占的） |

> **Workwriteの現状**: 適切。投稿者帰属 + 非独占的ライセンスは業界標準と一致。

## 2. Workwriteの現状評価

### 既に実装されているもの（強み）

#### AI追跡システム（業界最先端）
- `AiCreationLog`: 全AI生成アクションを段階別に記録（character_design, plot_architect, writing_assist等）
- `AiGenerationHistory`: AI会話履歴の完全保存
- **オリジナリティ算出**: `originality = 1.0 - (weightedAiChars / totalChars)`
  - 創作段階（キャラ設計等）は0.3x加重、本文執筆は1.0x加重
  - 50%超でisAiGenerated自動フラグ
- フロントエンド表示:
  - `AiGeneratedBadge` — AI生成作品バッジ
  - `OriginalityBadge` — オリジナリティ%表示（色分け: 90%↑緑, 70%↑青, 50%↑橙, 50%↓灰）
  - ダッシュボード・作品詳細・作品カード・管理画面で表示

#### 利用規約（既存）
- **Section 3**: 著作権は投稿者帰属、非独占的ライセンス
- **Section 4**: 盗作・無断転載・**AI生成コンテンツの未開示**を禁止行為に列挙
- **Section 6**: AI機能の説明 — 「作品データがAIモデルの学習に使用されることはありません」と明記
- プライバシーポリシー、特定商取引法に基づく表記も整備済み

#### レート制限
- AI補助: 20回/時/ユーザー
- 認証: IPベースで10回/15分（ログイン）、5回/15分（登録）
- リアクション: 30回/分/ユーザー

#### クレジットシステム
- AI機能はクレジット消費制（無限実行防止）
- 失敗時の自動返金
- トランザクション追跡（CONSUME/REFUND/PURCHASE等）

### 不足している項目（要対応）

#### 優先度: 高

**1. スクレイピング・自動収集の禁止規定**
- 利用規約に明文規定がない
- `robots.txt`が存在しない
- HTTPレベルのレート制限がない（認証済みユーザーのみ）
- ボット検出なし

**2. 重複投稿に関する方針の明記**
- 規約に重複投稿の可否が書かれていない
- 業界標準に合わせて「許可」と明記すべき

**3. AI生成コンテンツの開示義務の強化**
- 現在：禁止行為に「AI生成コンテンツの未開示」と記載あり
- 不足：具体的な開示基準（何%以上で開示必須か）が不明確
- 不足：開示しなかった場合の措置が不明確

**4. データエクスポート機能**
- プライバシーポリシーに「サポートへ連絡で対応」と記載
- 実装されたAPIエンドポイントは存在しない
- GDPR/個人情報保護法対応として自動エクスポート機能が望ましい

#### 優先度: 中

**5. 盗作・類似作品の検出**
- 作品間の類似度チェックが未実装
- コンテンツフィンガープリント（ハッシュ化）なし
- 外部盗作検出サービスとの連携なし

**6. コンテンツウォーターマーク**
- 作品テキストへの不可視マーカー埋め込みなし
- 無断転載時の追跡が困難

**7. DMCA対応・著作権侵害通報フロー**
- 著作権侵害の通報フォーム/窓口が規約に未記載
- テイクダウン手続きの明文化なし

#### 優先度: 低

**8. サイトマップ（sitemap.xml）**
- SEO面で不足

**9. Content Security Policy (CSP) ヘッダー**
- XSS防止の追加レイヤー

## 3. 規約改善案

### 追加すべき条項

#### 3-1. スクレイピング・自動収集の禁止（新規追加）

```
第X条（自動収集の禁止）
1. 当サービスのコンテンツ（作品テキスト、レビュー、ユーザー情報等）を、
   手動での閲覧以外の方法で収集・複製・蓄積する行為を禁止します。
   これには以下を含みますが、これに限りません：
   - スクレイピング、クローリング、ボットによる自動アクセス
   - APIの非公開エンドポイントへのアクセス
   - 当サービスのデータをAIモデルの学習データとして収集・利用する行為
2. 検索エンジンのクローラーは、robots.txtの指定に従う範囲で許可します。
3. 違反が確認された場合、IPアドレスのブロック及び法的措置を含む
   対応を行う場合があります。
```

#### 3-2. 重複投稿の許可（明示化）

```
第X条（他サービスとの重複投稿）
1. 投稿者は、他の小説投稿サービスで公開中の作品を本サービスにも
   投稿することができます（重複投稿）。
2. 重複投稿を行う場合、投稿者は当該作品の権利を有していること、
   または権利者の許諾を得ていることを保証するものとします。
3. 他サービスの利用規約により重複投稿が制限されている場合、
   投稿者はその規約を遵守する責任を負います。
```

#### 3-3. AI生成コンテンツの開示義務（強化）

```
第X条（AI生成コンテンツの取扱い）
1. 当サービスでは、AIを活用した創作を支持します。
   ただし、読者への透明性を確保するため、以下のルールを設けます。

2. 当サービスのAI機能を利用した場合：
   - AI利用率は自動的に計測され、作品ページに表示されます
   - 投稿者がこの表示を手動で変更・非表示にすることはできません

3. 当サービス外のAIツールを使用して本文を生成した場合：
   - 投稿者は、作品設定画面で「AI利用あり」を選択する義務があります
   - 本文の概ね50%以上がAI生成の場合、「AI主体の作品」として表示されます

4. AI生成の未開示が判明した場合：
   - 初回: 警告及びAIフラグの強制付与
   - 繰り返し: 作品の非公開措置またはアカウント停止

5. AI補助的利用（プロット検討・校正・表現の相談等）は開示不要です。
```

#### 3-4. 著作権侵害への対応（新規追加）

```
第X条（著作権侵害への対応）
1. 当サービスに投稿された作品が第三者の著作権を侵害している
   との通報を受けた場合、当社は以下の手順で対応します：
   a. 通報内容の確認
   b. 該当作品の一時非公開
   c. 投稿者への通知及び弁明の機会の付与
   d. 調査結果に基づく最終措置（削除・復元・アカウント停止）

2. 著作権侵害の通報は info@workwrite.co.jp 宛にお送りください。
   通報には以下を含めてください：
   - 権利者の氏名・連絡先
   - 侵害されている原著作物の特定情報
   - 当サービス上の該当作品のURL
   - 侵害の具体的な説明

3. 虚偽の通報を行った場合、通報者が法的責任を負う場合があります。
```

## 4. Workwrite AI安全性の整理（規約・説明ページ用素材）

### 「Workwriteにおける AI利用の安全性」

以下はWorkwriteのシステムで実際に実装されている制御の一覧です。規約やヘルプページに記載可能な事実です。

#### 作品データの保護
- **AIモデルの学習には一切使用しません** — Anthropic Claude APIはinference-only。Anthropicの利用規約上、APIデータはモデル改善に使用されません
- **作品テキストは外部に送信されません** — AI分析結果はWorkwriteサーバーに保存され、第三者と共有されません
- **AI機能は作品単位でON/OFF可能** — `enableCharacterTalk`等のフラグで作品ごとに制御

#### AI生成の透明性
- **全AI操作を段階別に記録** — `AiCreationLog`に生成・採用・却下・編集のすべてのアクションを保存
- **AI会話履歴の完全保存** — `AiGenerationHistory`に全メッセージ交換を記録
- **オリジナリティの自動算出** — AI生成文字数/総文字数から算出。創作段階（設計=0.3x）と本文執筆（1.0x）で加重を分け、補助的利用を過大評価しない
- **AI生成率50%超で自動フラグ** — `isAiGenerated`フラグが自動で付与され、作品ページに表示
- **オリジナリティバッジの常時表示** — 作品カード・詳細ページ・ダッシュボードで色分け表示（緑90%↑/青70%↑/橙50%↑/灰50%↓）

#### 不正利用の防止
- **クレジット制によるAI利用上限** — 無制限のAI生成は不可。月間付与+購入制
- **レート制限** — AI補助は20回/時/ユーザー
- **失敗時の自動返金** — AI呼び出し失敗時はクレジットを即座に返金
- **管理者によるAIフラグ制御** — 管理画面からisAiGeneratedの手動切り替え可能

#### 未実装（今後の課題）
- 盗作・類似作品の自動検出
- コンテンツウォーターマーク
- HTTPレベルのスクレイピング防止
- robots.txt
